## Sub-threshold observations (confidence < 70; recorded, not blocking)

1. **Alias contingency has no owner (~40).** Spec §10 permits satisfying a legacy path by *same path OR a Hugo `aliases` redirect*, and names case-mixed dirs as the risk. The plan's `verify-migration.py` enforces same-path existence (hard-binary, no adjudication), but includes **no task to add aliases** if a path diverges. Low probability because Hugo's deterministic lowercasing (verified above) makes same-path reproduction the expected outcome, and the CI verifier would surface any divergence as a red build rather than a silent miss — but there is no pre-planned remediation.
2. **Local verification runs on macOS; the case-sensitivity risk lives on Linux (~45).** Task 5's build + path check + visual compare run on the case-insensitive local FS, so a green Task 5 does not, by itself, prove the case-sensitive Linux build passes. This is owned by the Task 4 CI verifier before merge, so it is not silent — but the real §10 case gate is CI, not Task 5.
3. **Source freeze vs "update to latest" (~35).** Spec Phase A step 1 says update both local repos to latest `origin/master`; the plan instead pins the source to baseline `7c755e1` and asserts it. A deliberate, self-consistent reproducibility choice, not a drop.

## Pre-mortem (mandatory)

Single most likely missing/mis-sequenced step: **the Linux CI build (Task 4 workflow) surfaces a path-case divergence that macOS Task 5 could not, and the plan has no alias/remediation task for it — citing Spec §10 (URL 兼容性, case-sensitivity as a blocking condition).** The plan's local gate is case-insensitive and its verifier is a hard 73-path binary with no §10 alias fallback wired in, so if any page-bundle resource is referenced with a case that only resolves on macOS, the migration PR would stall at CI red with no pre-planned fix. This is low-probability (Hugo lowercases URL paths deterministically, and content images were verified to reference matching case) and, crucially, **not silent** — CI catches it before merge — which is why it stays below the reporting threshold rather than becoming a blocking finding.
