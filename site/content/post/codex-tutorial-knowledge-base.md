## 一、个人知识库不是“把文件放在一起”

**个人知识库，就是一座属于自己的数字书房。**

普通文件夹解决的是“文件放在哪里”；个人知识库还要解决三个问题：

- 我应该从哪里开始看？
- 这些资料之间有什么关系？
- 几个月以后，我还能不能快速找到需要的内容？

BL75 已经具备一个知识库的基本样子：

- `P-BL75生活攻略.md` 是入口，像书房门口的目录。
- `BL75-基础资讯.md` 和 `BL75-课程体系.md` 负责介绍背景。
- 两份功课题目负责保存练习。
- `BL75-自我指引工具.md` 负责保存可以反复使用的方法。
- `组会运行规则` 文件夹负责保存执行规则。

**告诉 Codex：**

> 先只读查看 BL75，不要修改文件。请用普通话告诉我：这个知识库的首页是哪一个文件，主要分成哪几类内容，每一类应该从哪个文件开始看。

**你要检查：** Codex 指出的首页是 `P-BL75生活攻略.md`，而且它提到的文件都真实存在。

## 二、什么是 Markdown

**Markdown 是一种带有简单标记的纯文字文件。**

它的文件名通常以 `.md` 结尾。你写下的内容仍然是普通文字，只需要在文字前后加几个简单符号，就能表示标题、列表、重点和链接。

在 BL75 里，你会看到四种最常用的写法：

- `# BL75-基础资讯`：`#` 表示标题。
- `- 个人成就 PA`：`-` 表示列表中的一项。
- `**负责任**`：两边的 `**` 会把文字加粗。
- `[[BL75-自我指引工具]]`：双中括号表示跳转到另一篇笔记。

你现在不需要背这些符号。Codex 可以替你整理格式，你只需要看得懂整理后的结果。

Markdown 最大的好处是：文件不属于某一个软件。即使以后不用 Obsidian，这些 `.md` 文件仍然可以用其他文字工具打开，也可以继续交给 AI 阅读。

**告诉 Codex：**

> 只读查看 `P-BL75生活攻略.md`，找出一个标题、一个列表和一个链接，分别告诉我它们在 Obsidian 里会显示成什么样。不要修改文件。

**你要检查：** Codex 给出的三个例子都能在原文件中找到，而不是它自己编出来的。

## 三、什么是 Obsidian

**Obsidian 是一个阅读、编辑和连接 Markdown 文件的软件。**

如果 Markdown 是一本本书，Obsidian 就是书架、目录和搜索台。它不会把你的笔记锁在自己的服务器里，主要内容仍然保存在电脑上的普通文件夹中。

Obsidian 把打开的知识库文件夹称为 **vault**，中文界面通常叫“仓库”。这里的“仓库”只是一个本地知识库文件夹，不是上一篇介绍的 GitHub 仓库。

在这套流程中，四个工具各做一件事：

- Markdown 保存内容。
- Obsidian 帮人阅读、搜索和跳转。
- Codex 帮你整理、检查和理解资料。
- Git 与 GitHub 负责记录版本和保存已经确认可以上传的副本。

个人知识库不一定要公开。只要里面有私人日记、账号信息或他人的隐私，就不要上传到公开 GitHub 仓库。

## 四、用 Obsidian 打开 BL75

先打开 [Obsidian 官方下载页](https://obsidian.md/download)，选择自己电脑对应的版本，像安装普通软件一样完成安装。

第一次打开 Obsidian 时：

1. 找到“打开本地文件夹”或“Open folder as vault”。
2. 点击旁边的“打开”。
3. 选择上一篇使用过的 `BL75` 文件夹。
4. 确认打开。

不同版本的按钮文字可能略有差别，只要选择的是“打开已有文件夹”，而不是“新建空白仓库”即可。

打开后先认识四个位置：

- 左侧文件列表：查看知识库里有哪些文件。
- 中间正文：阅读当前笔记。
- 左侧放大镜：搜索整个知识库。
- 左上方返回箭头：回到刚才阅读的笔记。

![](https://img.jdy.systems/tutorials/codex/bl75-obsidian-open-vault.webp)

*图：BL75 成功作为知识库打开后，左侧会显示课程、功课、工具和组会规则等文件。*

**你要检查：** 左侧能看到 `BL75-基础资讯.md`，点击后中间能显示“BL75生活攻略时间表”等正文内容。

如果左侧什么都没有，最常见的原因是选错了文件夹。重新选择那个直接包含 `BL75-基础资讯.md` 和其他课程文件的 BL75 文件夹。

## 五、从首页开始浏览，而不是在文件堆里乱翻

在左侧打开 `P-BL75生活攻略.md`。它是 BL75 的项目首页，也就是整个知识库的导航页。

以后找资料，可以按四条路线走：

1. **想了解背景**：打开 `BL75-基础资讯.md` 和 `BL75-课程体系.md`。
2. **想找课后练习**：打开 `BL75-MD功课题目.md` 或 `BL75-AT功课题目.md`。
3. **遇到问题想找方法**：打开 `BL75-自我指引工具.md`。
4. **准备活动或会议**：打开 `BL75-日常事务标准.md` 或 `组会运行规则` 文件夹。

页面中出现 `[[BL75-自我指引工具]]` 这样的链接时，点击它就能进入对应笔记，再用返回箭头回到首页。

**告诉 Codex：**

> 只读分析 `P-BL75生活攻略.md` 的导航是否完整。逐项检查它链接的文件是否存在，并给我一条适合第一次阅读 BL75 的顺序。不要修改任何文件。

**你要检查：** Codex 没有报告失效链接；建议顺序至少包含基础资讯、课程体系、自我指引工具和日常事务标准。

## 六、用搜索找到散落在不同文件里的内容

目录适合“我知道要找哪一类内容”的情况。搜索适合“我只记得一个词”的情况。

点击 Obsidian 左侧的放大镜，先搜索 `负责任`。你会看到这个词不只出现在一个地方：基础资讯、自我指引工具和部分规则都可能提到它。

再搜索 `小组会`，就能同时找到课程标准和团队制定的运行规则。

这正是知识库比单篇文档更有用的地方：同一个概念可以出现在不同资料中，搜索把它们重新聚到一起。

**你要检查：** 点击任意一条搜索结果后，正文会打开包含这个词的真实位置，而不是只显示文件名。

## 七、让 Codex 基于知识库回答问题

搜索负责帮你找原文，Codex 可以进一步帮你阅读、比较和解释。但要给它一条重要规则：**回答必须有来源。**

**告诉 Codex：**

> 只根据 BL75 文件夹中的内容回答，不要使用你自己的常识补充。问题是：当我遇到一件事，想马上凭感觉作出反应时，BL75 里有什么工具可以帮助我？请用普通话解释，并注明答案来自哪个文件、哪个小节。如果知识库里没有答案，直接告诉我没有找到。

一个可靠的回答应该指向 `BL75-自我指引工具.md` 的“工具三：停、看、选择、行动”，并解释先停止自动反应，再观察、选择和行动。

不要因为回答听起来合理就直接相信。回到 Obsidian，打开 Codex 指出的文件和小节，确认原文真的支持它的说法。

**你要检查：** 文件名、小节名称和核心步骤都能在原文中对应上。找不到对应内容，就继续追问 Codex 来源在哪里。

可以长期复用这句话：

> 回答这个问题时，只能使用当前项目里的资料。每个结论都要注明来源文件和小节；没有资料支持的部分写“知识库中没有找到”，不要猜测。

## 八、让 Codex 帮你整理一份新资料

知识库不是一次整理完就结束。真正有用的流程是：获得新资料后，知道放在哪里，并且让旧资料能够找到它。

以后拿到一份新笔记、会议安排或学习材料，可以先这样说：

**告诉 Codex：**

> 先只读查看 BL75 的现有文件和导航，不要修改。分析我接下来提供的新资料更适合归入“背景与课程”“功课”“自我指引工具”还是“运行规则”。告诉我推荐的文件名、保存位置、应该链接到哪一篇现有笔记，以及你的理由。先等我确认。

把资料交给 Codex 后，先看它的建议，不要立刻让它写文件。

**你要检查：** 文件名能说明内容；位置与同类资料一致；至少有一个现有入口能够链接到它；Codex 没有准备修改无关文件。

确认方案后，再告诉它：

> 我确认这个整理方案。请把新资料整理成 Markdown，只修改刚才确认的文件，并补上必要的导航链接。完成后列出修改了哪些文件，用普通话总结每项变化。不要提交，也不要上传。

修改完成后，在 Obsidian 里做三次检查：

1. 新笔记能正常打开。
2. 从首页或相关笔记能够点击进入新笔记。
3. 搜索一个新笔记里的关键词，能够找到它。

这三项都通过，才算真正进入知识库。只有文件存在、却没有入口也搜索不到，它仍然只是被塞进了文件夹。

## 九、最简单的长期维护方法

不需要一开始就设计复杂的分类系统。先守住一个小闭环：

1. 新资料先收进来。
2. Codex 帮你判断放在哪里。
3. 你确认内容和位置。
4. Codex 整理格式并建立链接。
5. 你在 Obsidian 中打开、点击和搜索验证。
6. 确认没有隐私后，再决定是否用 Git 保存版本、是否上传 GitHub。

知识库的价值不在于文件数量，而在于需要的时候能找到，新增内容时知道放哪里。
