## 一、为什么必须梯子

OpenAI 当前的 ChatGPT 支持地区列表中不包含中国大陆、香港和澳门。官方同时提醒：从列表以外的地区访问或提供访问，可能导致账号被封禁或暂停。

这一篇介绍网络连接和排查方法，不代表 OpenAI 对未支持地区作出了服务承诺。继续之前，请自行确认符合所在地法律以及你愿意接受的账号风险。

如果你决定继续，技术上需要让电脑通过一个支持地区的网络出口连接 OpenAI。完成这件事的工具，通常叫**代理**，也常被称为“梯子”或“科学上网”。

先立一个预期：**只要在用 Codex，梯子就得一直开着**，不是登录时开一下就关。所以这一篇的目标不是"翻出去一次"，而是搭一条**长期稳定**的线路。

## 二、Codex 对梯子的三个特殊要求

Codex 桌面 App 会持续访问网络。下面三个要求不能保证一定通过 OpenAI 风控，但能减少常见的连接问题。

### 要求 1：IP 要"干净"（最重要）

挂上梯子后，OpenAI 看到的是**梯子出口的 IP**（相当于你上网的"门牌号"）。如果这个 IP 之前被人拿去干过坏事（群发垃圾、批量注册），就会进风控黑名单——这叫"脏 IP"。用干净 IP，注册登录一路顺畅；用脏 IP，轻则反复弹验证、被要求手机验证，重则封号。

多人共用一个出口 IP 时，其中任何人的异常行为都可能影响其他使用者。可以把它想成合租房的门牌号：前一个住户留下问题，后面的住户也可能被重点检查。

**辅助检测方法**：挂上代理后打开 [ping0.cc](https://ping0.cc/)，查看出口 IP 的归属地和风险提示。它是第三方筛查工具，不是 OpenAI 官方检测；“低风险”只能作为辅助判断，不能保证账号一定通过风控。

### 要求 2：节点地区在支持列表内就行

美国、日本、新加坡、英国、韩国、台湾等都可以（[官方支持列表](https://help.openai.com/en/articles/7947663-chatgpt-supported-countries)）。**不能用香港、澳门**——和大陆一样不提供服务，而且很多梯子的默认节点恰恰是香港，这是最容易踩的坑。

节点地区只要在官方支持列表内即可。遇到登录失败，可以先换一个支持地区的节点测试；如果换节点后恢复，通常说明问题与原节点的线路或出口 IP 有关，而不是某个国家一定不能用。

### 要求 3：梯子要接管"整台电脑"，不能只管浏览器

Codex 桌面 App 是独立程序，流量**不经过浏览器**。如果梯子只配置在浏览器里，就会出现：浏览器能开 chatgpt.com，Codex App 却死活连不上。就像净水器只装在厨房水龙头（浏览器），浴室的水（桌面 App）还是生水——你需要**全屋净水**。

客户端里这个模式叫 **TUN 模式**（也叫"虚拟网卡"、"增强模式"），第五节实操会教你打开。

## 三、已经有梯子？先做三步自检

三项全过，**这一篇你就毕业了，直接去第 2 篇**。

| 自检       | 怎么做                           | 过关标志            |
| -------- | ----------------------------- | --------------- |
| ① 连通性    | 挂梯子，浏览器打开 chatgpt.com         | 页面正常显示（能看到登录界面） |
| ② IP 辅助检查 | 打开 ping0.cc                   | 归属地在支持列表内，未出现明显高风险提示  |
| ③ 全局能力   | 在梯子客户端里找「TUN / 虚拟网卡 / 增强模式」开关 | 找得到，打开后 ① 依然过关  |

- ① 过 ② 不过 → 换个节点再测；所有节点都脏 → 换一条线路（看第四节）。
- ③ 不过 → 换客户端就行（第五节装 Clash Verge，节点不用换）。

## 四、还没有梯子：找我，两条路

| 维度   | 机场（买现成服务）      | 自建 VPS（租服务器自己搭）        |
| ---- | -------------- | ---------------------- |
| 门槛   | 低：付钱拿链接导入      | 要动一次手，30–60 分钟         |
| 月成本  | 常见 ¥10–50      | 独享 ¥100 左右；拼车人均几十块     |
| IP   | 多人共享，历史使用情况难判断   | 使用者更少，出现问题时更容易排查             |
| 适合谁  | 想今天就用上、不想折腾    | 打算长期用、要最稳              |

**机场**：市面上鱼龙混杂，跑路是常态，我不在教程里公开点名。**想走这条路的直接私聊我**，我推荐几个自己用过的。一条纪律带走：**永远月付，别囤年费**——商家跑路无法维权，月付把最大损失锁死在一个月。

**自建 VPS**：长期用 Codex 最稳的方案——IP 自己人独享，不看任何商家的良心。两个选择：

1. **找我要教程，自己搭**。适合愿意动手的：购买服务器后照教程完成一次配置，不需要先学会编程。
2. **委托我搭**。不想碰命令行可以选这个。几位熟人共同使用能分摊成本，也更容易知道线路由谁使用；但共享线路仍然不能保证账号一定不触发风控。

无论哪条路，你最终会拿到一条**订阅链接**（一串 `https://` 开头的网址），接着往下做。

## 五、实操：用 Clash Verge 把电脑连上梯子

客户端统一用 **Clash Verge**（全名 Clash Verge Rev）：免费开源、Mac / Windows 都有、支持 TUN、对新手友好。

### 5.1 下载安装

只从官方发布页下载，**别搜"Clash 下载"点进不明网站**——山寨安装包是真实存在的投毒渠道：

- 官方发布页：[Clash Verge Rev Releases](https://github.com/clash-verge-rev/clash-verge-rev/releases/latest)
- Mac 选 `.dmg`（Apple 芯片选 aarch64，Intel 芯片选 x64）；Windows 选 `x64-setup.exe`

> 没有梯子时 GitHub 下载可能很慢甚至失败：换个网络重试（比如手机热点），或者直接找我拿安装包。

安装：Mac 双击 `.dmg` 把图标拖进"应用程序"；Windows 双击安装包一路下一步。

### 5.2 导入订阅

1. 打开 Clash Verge，左侧进入「订阅」页
2. 把订阅链接粘贴进顶部输入框，点「导入」
3. 导入成功会出现一张配置卡片，**单击卡片激活**（卡片边缘出现高亮）

![](https://img.jdy.systems/codex-tutorial/clash-verge-subscription-import.webp)

### 5.3 打开 TUN 模式（关键一步）

1. 「首页」的「网络设置」卡片里，点选「虚拟网卡模式」并打开下方开关（旧版本在「设置」页，叫「TUN 模式」）
2. 第一次打开会要求安装系统服务：Mac 弹出输密码或"允许后台项目"，Windows 要管理员权限——都同意

成功标志：出现「TUN 模式已启用，应用将通过虚拟网卡访问网络」的提示：

![](https://img.jdy.systems/codex-tutorial/clash-verge-tun-mode.webp)

### 5.4 选节点

左侧「代理」页，在「节点选择」分组里点中你要用的节点（机场用户会看到一长串，挑支持列表内地区、延迟低的）：

![](https://img.jdy.systems/codex-tutorial/clash-verge-select-node.webp)

顺便认识页面右上角的三个挡位：

- **规则**（默认，日常用这个）：智能分流——国内网站直连不绕路，国外流量才走梯子
- **全局**：所有流量一律走梯子，不分国内外；国内网站会变慢，但最不容易出错
- **直连**：全部不走梯子，相当于临时关掉

平时保持「规则」就好；遇到"梯子开着却连不上"，临时切「全局」测试——全局能通而规则不通，说明是分流规则的问题，找给你线路的人处理。

### 5.5 验证

- 浏览器打开 `chatgpt.com` → 能正常显示页面
- 打开 `ping0.cc` → 归属地是你选的节点地区，且没有明显高风险提示（仅作辅助判断）

## 六、做完的标志

三项全绿，再进第 2 篇：

- [ ] 浏览器能稳定打开 chatgpt.com
- [ ] ping0.cc 显示归属地在支持列表内，且没有明显高风险提示
- [ ] TUN 模式开着，且上面两条依然成立

**连不上？按顺序排查三步**：① 换节点（最常见原因就是这个 IP 被玩坏了）；② 查状态（Clash Verge 在运行？配置卡片激活了？TUN 真的开了？），再临时切「全局」模式试试——能通就是分流规则的问题；③ 换无痕窗口再试（排除浏览器缓存记住了旧位置）。还不行，直接来找我。
